# 证据优先级与冲突处理

在企业内部使用合规预警中，优先使用官方、可定位到版本、可追溯日期的来源。

## 证据优先级

按以下顺序采信：

1. 仓库中的 `LICENSE`、`COPYING`、`NOTICE`、发布包内许可证文本
2. 官网的 License、Licensing、Pricing、Commercial License、Billing、Plans、FAQ 页面
3. 官方发布说明、版本公告、博客中的 relicensing / licensing change 公告
4. 包管理器注册页中的许可证元数据
5. 社区讨论、博客、问答站点、第三方法律解读

## 冲突处理规则

- 官方许可证文本与博客冲突时，以官方许可证文本为准
- 官方价格页、Billing 页与第三方转载价格冲突时，以官方页面为准
- 官网 FAQ 与仓库根目录 `LICENSE` 冲突时，优先确认是否存在版本差异、双许可证或附加商用条款
- 只有第三方来源、没有官方来源时，结论默认倾向 `信息不足`
- 未标明版本的“可商用”说法，不能直接用于目标版本结论
- 未标明时间或地区口径的“价格/折扣”说法，不能直接用于采购结论

## 日期与版本要求

- 尽量记录页面发布时间、更新时间或 release 日期
- 对价格、折扣、套餐信息，尽量记录页面更新时间与地区/币种口径
- 尽量写清“适用版本”，不要把旧版本规则套到新版本
- 对曾发生过改 license 的项目，必须明确说明“哪一段版本适用哪种许可证”
