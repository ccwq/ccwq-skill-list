# 授权与付费整理规则

用于补充企业内部使用场景下的软件授权、公开价格与采购方式整理。
这是采购信息归纳，不是报价承诺，也不是法律意见。

## 适用触发条件

只要满足以下任一条件，就应输出“授权与付费整理”：

- 官方要求商业授权
- 存在商业版、企业版或订阅计划
- 存在公开 pricing / billing / plans 页面
- 存在月付、年付、按量、按席位、按设备、一次性授权等计费方式
- 存在公开折扣、年付优惠或促销信息
- 官网仅提供 sales 入口或 enterprise quote

## 证据优先级

按以下顺序采信：

1. 官网 Pricing、Plans、Billing、Licensing、Commercial License 页面
2. 官网 FAQ、Help Center、采购说明、销售说明页面
3. 官方博客、发布说明、版本说明中的价格或授权变更公告
4. 官方应用商店、官方文档中的套餐或结算说明
5. 权威二手来源，例如官方合作 reseller、可信文档镜像

如果只有第三方博客、论坛、社区转述，没有官方或权威二手来源，默认不写具体价格，最多写“存在相关说法，但证据不足”。

## 输出口径

- 默认按中国团队采购视角写
- 若官网价格是美元或美国站口径，要明确写“美元官网口径”
- 若能确认中国区价格、税费、发票或购买路径，可以补充
- 若无法确认中国区差异，不要把美国官网价格写成中国实际采购价

## 价格与方案写法

- `公开价格`
  仅写官网或高可信来源明确展示的价格
- `公开付费方式`
  需要区分月付、年付、按量、按席位、按设备、一次性授权、企业询价
- `公开折扣`
  只写官方明确可见的折扣，例如年付优惠、教育折扣、非营利折扣、限时活动
- `企业询价`
  若官网只写“Contact sales”或类似表述，应明确记为“有商业路径，但价格未公开”

## 禁止事项

- 不要根据历史博客、社区截图或他人经验猜测当前价格
- 不要把“联系销售”自动写成高风险
- 不要把按量计费混写成固定月费
- 不要把折扣传闻写成公开折扣
- 不要输出私下谈判策略、压价技巧或未经证实的企业报价区间

## 推荐表达

- 若存在公开价格：
  “官网公开提供月付和年付方案，年付较月付有公开优惠，当前看到的是美元官网口径。”
- 若只有企业询价：
  “官方明确存在企业采购路径，但官网未公开报价，采购前需要向销售确认席位数、部署方式和合同周期。”
- 若未发现付费要求：
  “当前未发现公开的额外付费要求，但仍建议在采购或长期依赖前保留官网条款截图。”
